1. Gitとは？ — タイムマシンのようなもの

Gitは「分散型バージョン管理システム」です。簡単に言うと、ファイルやコードの変更履歴を、まるでゲームのセーブデータのように、細かく記録・管理するためのツールです。

Gitを使う最大のメリット：

  • 過去に戻れる: いつでも過去の完璧な状態に戻すことができます。
  • 共同開発が簡単: 複数人で作業しても、誰がいつどこを変更したかが一目瞭然で、ファイルを安全に統合できます。
  • 実験が自由: 「ブランチ」という機能で、本番コードに影響を与えずに新機能のテストや開発ができます。

2. セットアップと初期化

2-1. ユーザー設定（最初の一回だけ）

コミット（履歴の記録）を行う際に、誰が変更したかを記録するために名前とメールアドレスを設定します。

$ git config --global user.name "あなたの名前" $ git config --global user.email "your.email@example.com"

2-2. リポジトリの作成（プロジェクトごと）

プロジェクトフォルダ内でこのコマンドを実行すると、Gitによる管理が開始されます。これでフォルダが「リポジトリ」になります。

$ cd /path/to/your/project $ git init

（`git init`でGit管理が始まります。）

3. コミットまでの基本フロー

Gitの作業は、以下の3つのステップで進みます。

Step 1: 変更

ファイルを編集、作成します。

Step 2: ステージング（変更を準備）

記録したい変更を一時エリア（インデックス）に登録します。

Step 3: コミット（履歴を記録）

ステージングした変更を確定させ、正式な履歴として保存します。

よく使うコマンド

変更の確認: どのファイルが変更されたかを確認します。
$ git status
ステージング: 全ての変更をステージングエリアに追加します。
$ git add .
コミット: 変更を確定し、履歴に記録します。コミットメッセージは重要です！
$ git commit -m "機能追加: ログイン機能の実装"

4. 履歴の確認と取り消し

4-1. 履歴の確認

これまでの全てのコミット履歴を一覧表示します。

$ git log $ git log --oneline --graph

4-2. 最新コミットの取り消し（メッセージ修正など）

直前のコミットの内容（ファイルやメッセージ）を修正したい場合に使います。

$ git commit --amend

（これにより、コミットメッセージの編集画面が開きます。）

5. リモートリポジトリとの連携

自分のPC（ローカル）の履歴を、GitHubなどの外部サーバー（リモート）と同期させます。

リモートの登録: リモートリポジトリの場所を登録します。（通常は`origin`という名前で登録します。）
$ git remote add origin https://github.com/user/repo-name.git
プッシュ (Push): ローカルの変更をリモートに「アップロード」します。
$ git push origin main
プル (Pull): リモートの最新の変更をローカルに「ダウンロード」して統合します。
$ git pull origin main

6. ブランチ操作とマージ

ブランチは、開発を枝分かれさせて独立して行うための機能です。新しい機能開発やバグ修正は、必ずメインのブランチから分けて行います。

ブランチ操作コマンド

ブランチの確認: 現在存在するブランチの一覧と、今いるブランチ（`*`が付く）を確認します。
$ git branch
ブランチの作成: 新しいブランチを作成します。（まだ移動はしません）
$ git branch feature/new-feature
ブランチの切り替え: 作成したブランチへ移動します。
$ git checkout feature/new-feature
マージ (Merge): 開発が終わり、メインのブランチ（例: `main`）に戻って変更を統合します。
$ git checkout main $ git merge feature/new-feature

これでGitの基本はマスターです！

最も重要なのは「変更 → ステージング → コミット」の繰り返しです。毎日少しずつ使って慣れていきましょう。
